Vous vivez dans la Matrice : un Bearer Token (access_token) vous donne l’accès à vos applications… 
mais si un Agent Smith l’intercepte ? Game over. 
Il a tous vos accès sans demande de mot de passe, pas de preuve de possession – vous avez donc juste une illusion de contrôle.

Suivez le lapin blanc jusqu’à DPoP (Demonstration of Proof-of-Possession, RFC 9449) : vos tokens deviennent inséparables d’une clé privée. 
Prouvez que vous êtes L’élu, ou rien.

Restons au pays des merveilles, et descendons avec le lapin blanc au fond du gouffre :
* PKCE, la première barrière contre l’interception du code d’autorisation
* Les failles critiques des bearer tokens
* Comment DPoP fonctionne
* Les pièges d’implémentation dans le monde réel où les Agents se cachent encore

… pour montrer qu’il est temps de sortir de l’illusion que toute la sécurité est faite avec les bearer tokens.
